Data

We will look at daily bike rentals from citibike, a bike share program in New York City, over the summer of 2019 and the summer of 2020.

citibike <- read_csv("citibike.csv")

 Column specification 
cols(
  date = col_date(format = ""),
  day = col_double(),
  year = col_double(),
  day_of_the_week = col_character(),
  rides = col_double()
)
# Data aggregated from https://www.citibikenyc.com/system-data

# the code below will make sure that the days of the week go in order rather than alphabetically,
# and that year is treated not as a number but as a categorical variable
citibike <- citibike %>%
  mutate(day_of_the_week = factor(day_of_the_week, levels = c("Mon", "Tue", "Wed", "Thu", "Fri", "Sat", "Sun")),
         year = factor(year))

# A dataset with only 2019 data and one with only 2020 data

citibike2019 <- citibike %>%
  filter(year == "2019")

citibike2020 <- citibike %>%
  filter(year == "2020")

2020 model fit

Now we will look at 2020 only. We will include a day variable, which is day of the year, and also separate intercepts for the day of the week, using a hierarchical model. The argument control = list(adapt_delta = 0.999) is something to fix a problem that we wont discuss for a few weeks.

citi_hier_2020 <- stan_glmer(rides ~ day + (1 | day_of_the_week) , data = citibike2020, control = list(adapt_delta = 0.999))

Look at results

Lets take a look at the distribution of the day of the week specific changes in the intercept. The regex_pars = "Intercept.+day.of.the.week" argument below is a pattern designed to pick out just the parameters we wantwell talk about it just a little later this semester. If youre curious, . represents any character, and .+ represents one or more of any character.

plot(citi_hier_2020, plotfun = "areas_ridges", regex_pars = "Intercept.+day.of.the.week")

Pretty clearly Saturday and Sunday show a shift towards higher rides.
